Build the full T&C + Customer Acceptance flow for Project/Install estimates. Here is the complete spec:

## 1. T&C MANAGEMENT (DB + Admin Settings)

Create a new DB table terms_and_conditions: id, type (TEXT: 'install' | 'maintenance' | 'snow'), title (TEXT), content (TEXT), is_active (BOOLEAN default true), created_at, updated_at

Seed it with the Install T&C (type='install', title='Install Project Terms & Conditions').

Add a Settings page at /settings/terms (linked from the existing Settings nav) that lets admin:
- View all T&C types (Install, Maintenance, Snow tabs)
- Edit the content of each in a large textarea
- Save changes (PUT /api/settings/terms/:id)

API routes: GET /api/settings/terms, PUT /api/settings/terms/:id (admin only)

## 2. AUTO-POPULATE T&C ON ESTIMATES

Add column terms_and_conditions_override TEXT to sales_estimates.

On the estimate detail page, add a collapsible "Terms & Conditions" section at the bottom that shows:
- The T&C text (from override if set, otherwise from the active install T&C for that job type)
- "Edit Terms" button opens a textarea to customize for this estimate
- "Reset to Default" clears the override

## 3. CUSTOMER PORTAL - T&C + INITIALS + SIGNATURE + ACCEPT

Update the customer portal (GET /api/portal/:token) to include full acceptance flow:

Section order:
1. Chapin Landscapes logo + estimate header
2. Scope/line items summary
3. Terms & Conditions - scrollable box, customer must scroll to bottom before initials activates
4. Initials field - text input "Type your initials to confirm you have read and agree to the Terms & Conditions above" - required
5. Signature pad - canvas-based (no external library, plain canvas draw)
6. Accept button - green, full width "Accept & Approve Proposal" - disabled until initials filled AND signature drawn
7. Decline button - secondary "Decline Proposal"

On Accept: POST /api/portal/:token/respond with { action: 'approved', initials, signature_data (base64 canvas), note: '' }
Show confirmation: "Thank you [customer name]! Your approval has been confirmed."

On Decline: Show textarea "Why are you declining? (optional)" then POST with { action: 'declined', note }

## 4. APPROVAL NOTIFICATION

When approved:
- Update estimate status to 'approved', set approved_at = NOW()
- Create notifications table if not exists: id, user_id, message, estimate_id, read BOOLEAN default false, created_at
- Insert notification for assigned user (or all admins): "Customer [name] approved estimate #[number] - [title]"
- Bell icon in top nav shows unread count badge and lists notifications on click

## 5. AUTO DOWN PAYMENT INVOICE

When estimate is approved:
- Auto-create invoice: estimate_id, customer_id, type='down_payment', amount = total * 0.5, status='pending', due_date = NOW() + 7 days, description = "50% Deposit - [estimate name]"
- Store deposit_percentage column on sales_estimates (default 50) so it's configurable per estimate
- Invoice appears in Invoices section linked to the estimate
