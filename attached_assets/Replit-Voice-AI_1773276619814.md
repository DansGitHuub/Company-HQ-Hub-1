# Replit Prompt — CompanyHQ AI Assistant Voice

Paste everything below this line into Replit Agent:

---

Add full voice capability to CompanyHQ's AI Assistant. This includes speech-to-text (user speaks to the AI) and text-to-speech (AI speaks responses back). Voice should work globally across the entire app on both desktop and mobile, with per-user settings to control behavior.

---

## PART 1 — USER VOICE SETTINGS

Add voice preference fields to the users table if not already present:
- voice_enabled (boolean, default false) — master on/off for voice features
- voice_auto_speak (boolean, default false) — automatically speak AI responses
- voice_selection (text, default "alloy") — which voice the AI uses to speak

Add a Voice section to the user's Profile/Settings page with:
- **Voice Features** toggle — master on/off switch for all voice features
- **Auto-speak responses** toggle — when on, AI responses are read aloud automatically. When off, user can click a speaker icon on any response to hear it
- **AI Voice** selector — let user pick from these OpenAI TTS voices with a preview button next to each:
  - Alloy (neutral, balanced)
  - Echo (male, conversational)
  - Fable (male, expressive)
  - Onyx (male, deep)
  - Nova (female, warm)
  - Shimmer (female, clear)
- A **Preview Voice** button that speaks a short sample sentence using the selected voice
- Save button — saves all voice settings to the user's record via PATCH /api/users/voice-settings

---

## PART 2 — GLOBAL MIC BUTTON

Add a global floating microphone button to the app layout. It should:
- Appear in the bottom right corner of the screen on all pages
- Be visible on both desktop and mobile
- Only show if the user has voice_enabled = true in their settings
- Be a round button with a microphone icon
- Have three visual states:
  - **Idle** — gray/neutral, mic icon
  - **Listening** — red pulsing animation, mic icon, "Listening..." tooltip
  - **Processing** — spinner animation while sending to AI
- Clicking it from any page opens the AI Assistant in a slide-in panel (if not already open) and activates the microphone

---

## PART 3 — SPEECH TO TEXT (MIC INPUT)

Use the browser's built-in Web Speech API (SpeechRecognition) for speech-to-text. This works on Chrome, Edge, and mobile browsers with no API key needed.

Implementation:
- When mic button is clicked: request microphone permission if not already granted
- Start SpeechRecognition with:
  - continuous: false
  - interimResults: true
  - language: 'en-US'
- While listening: show a live transcript preview in the chat input field in gray italic text
- When speech ends: finalize the transcript, place it in the chat input as normal text
- Automatically submit the message to the AI (same as pressing Enter)
- If microphone permission is denied: show a friendly message explaining how to enable it in browser settings
- If Web Speech API is not supported (Firefox): show a message saying voice input requires Chrome, Edge, or Safari

Also wire up the existing mic button in the AI Assistant chat panel if one already exists — connect it to this same SpeechRecognition logic.

---

## PART 4 — TEXT TO SPEECH (AI SPEAKS RESPONSES)

Use the OpenAI TTS API to generate spoken audio from AI responses.

**Backend route — POST /api/ai/speak**

Accepts:
- text (string) — the text to speak
- voice (string, optional) — override voice, defaults to user's voice_selection setting

Process:
- Call OpenAI Audio API: POST https://api.openai.com/v1/audio/speech
- Model: tts-1
- Voice: user's selected voice (alloy, echo, fable, onyx, nova, shimmer)
- Input: the text to speak
- Response format: mp3
- Stream the mp3 back to the client

Use the existing OPENAI API key from Replit Secrets (AI_INTEGRATIONS_OPENAI_API_KEY).

**Frontend playback:**

When an AI response arrives in the chat:
- If auto_speak is ON: automatically call /api/ai/speak with the response text and play the audio
- If auto_speak is OFF: show a small speaker icon button on each AI message bubble. Clicking it calls /api/ai/speak and plays that message
- While audio is playing: the speaker icon animates (sound wave animation)
- Clicking the speaker icon again while playing stops the audio
- Only one audio clip plays at a time — starting a new one stops the previous

For very long responses: speak only the first 500 characters to keep it natural. Add "..." if truncated.

## PART 5 — VOICE SETTINGS QUICK TOGGLE

In the AI Assistant panel header, add two small icon buttons:
- **Mic toggle** — enables/disables voice input for this session (microphone icon, highlights when active)
- **Speaker toggle** — enables/disables auto-speak for this session (speaker icon, highlights when active)

These are session-level overrides — they don't change the saved settings in the user profile, just toggle behavior for the current session. Tooltips should say "Voice input on/off" and "Auto-speak on/off".

The initial state of these toggles should reflect the user's saved voice_auto_speak setting.
