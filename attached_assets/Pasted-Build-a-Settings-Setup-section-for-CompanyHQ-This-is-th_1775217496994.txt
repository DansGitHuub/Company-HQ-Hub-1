### TAB 1: Work Areas (/settings/work-areas)

Manages the work_area_types table (id, name, division, is_active, sort_order).

Display a table with columns: Name, Division, Active (toggle), Sort Order, Actions (Edit/Delete).
Group rows by division with a sticky group header.
"+ Add Work Area" button opens an inline form or modal:
  - Name (text input)
  - Division (select: Maintenance | Install | Snow | General)
  - Active (checkbox, default true)
  - Sort Order (number)
  - Save / Cancel

Edit: click pencil icon → inline edit row or same modal pre-filled.
Delete: click trash icon → confirm dialog "Delete [name]? This cannot be undone."
Toggle Active: click toggle directly in table row.

API routes needed:
GET /api/work-area-types — already exists, use it
POST /api/work-area-types — create new (check if exists, add if missing): { name, division, is_active, sort_order }
PUT /api/work-area-types/:id — update: { name, division, is_active, sort_order }
DELETE /api/work-area-types/:id — delete (only if not in use by any job_work_areas)

Manage the divisions used throughout the app (for color-coding jobs, estimates, scheduling).

For now this can be a STATIC editable list stored in a new app_settings table or just hardcoded as a display with color swatches. Show:
- Maintenance · #22c55e (green swatch)
- Install · #3b82f6 (blue swatch)
- Snow · #94a3b8 (slate swatch)

Each row: color swatch, name, hex color input. Allow admin to change the hex color.
Store in a new app_settings table:

Seed with: division_colors = '{"Maintenance":"#22c55e","Install":"#3b82f6","Snow":"#94a3b8","General":"#f59e0b"}'

API: GET /api/settings/:key and PUT /api/settings/:key (value in body)

## 4. MIGRATIONS — server/migrations/appSettings.ts

Create this migration:

export async function runAppSettingsMigration() {
  const client = await pool.connect();
  try {
    await client.query('BEGIN');
    await client.query(`
      CREATE TABLE IF NOT EXISTS app_settings (
        key VARCHAR(100) PRIMARY KEY,
        value TEXT,
        updated_at TIMESTAMP DEFAULT NOW()
      )
    `);
    // Seed division colors
    await client.query(`
      INSERT INTO app_settings (key, value) VALUES ('division_colors', '{"Maintenance":"#22c55e","Install":"#3b82f6","Snow":"#94a3b8","General":"#f59e0b"}')
      ON CONFLICT (key) DO NOTHING
    `);
    // Seed company info placeholder
    await client.query(`
      INSERT INTO app_settings (key, value) VALUES ('company_info', '{"name":"Chapin Landscapes","phone":"","email":"","address":"","website":"","tax_rate":"0","payment_terms":"Net 30"}')
      ON CONFLICT (key) DO NOTHING
    `);
    await client.query('COMMIT');
  } catch (err) {
    await client.query('ROLLBACK');
    throw err;
  } finally {
    client.release();
  }
}

Register this in server/migrations/index.ts following the same pattern as other migrations.
