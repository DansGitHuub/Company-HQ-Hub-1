"Build the complete Document Management System for CompanyHQ. This is a foundational feature that touches every module. Read everything before writing any code.
CORE PRINCIPLE: Every document has one home record where it was created or uploaded, but can be linked to multiple related records without duplicating the file. A customer's signed contract lives in the Job record AND appears in their Customer record — same file, two access points, zero duplication.
STEP 1 — DATABASE SCHEMA
Create these tables using Drizzle ORM:
documents: id, file_name, file_url (stored in Replit object storage or similar), file_type, file_size_kb, category (enum: form, policy, manual, registration, insurance, certification, photo, contract, proposal, invoice, warranty, report, other), uploaded_by_user_id, home_entity_type (enum: employee, equipment, job, customer, company), home_entity_id, description, is_template (boolean — for master form templates), version (integer default 1), created_at, updated_at.
document_links: id, document_id, linked_entity_type, linked_entity_id, linked_by_user_id, linked_at. This is what allows one document to appear in multiple places without duplication.
document_share_links: id, document_id, share_token (UUID, unique), created_by_user_id, expires_at, password_hash (nullable), note (nullable), view_count (default 0), is_revoked (boolean default false), created_at.
document_share_access_log: id, share_token_id, accessed_at, ip_address.
form_submissions: id, form_type (enum: w4, i9, ohio_it4, direct_deposit, handbook_acknowledgment, emergency_contact, background_check_auth, workers_comp_first_report, osha_incident, nda, employment_application), employee_id, submitted_by_user_id, submission_data (JSON — all field values), status (enum: draft, submitted, pending_review, approved, rejected), pdf_document_id (nullable — populated when PDF is generated), submitted_at, reviewed_by_user_id, reviewed_at, review_notes.
STEP 2 — DOCUMENTS TAB ON EVERY RECORD
Add a Documents tab to: Employee Records, Equipment Records, Job Records, Customer Records. Each tab shows all documents where home_entity matches that record PLUS all documents linked to that record via document_links. Display as a list with: file name, category badge, file type icon, upload date, uploaded by, file size, and action buttons (View, Download, Share Externally, Link to another record). Upload button at the top. Drag and drop upload supported. Show empty state with upload prompt when no documents exist. Role permissions: Admin and Manager can upload, view, download, share, and delete any document. Crew Lead can upload and view documents on equipment assigned to them. Crew Member can view their own employee documents only. Customers can view documents in their Customer record that are marked customer-visible.
STEP 3 — MASTER DOCUMENT LIBRARY IN ADMIN PANEL
Add a Documents section to the Admin Panel with two sub-tabs:
Company Library: Documents where home_entity_type is company. This is where company-wide documents live — employee handbook, NDA template, OSHA postings, company insurance certificates, company certifications, master form templates. Admin can upload, organize into folders by category, and manage versions. When a new version of the handbook is uploaded, the old version is archived not deleted.
All Documents: A searchable index of every document in the entire system regardless of where it lives. Search by file name, category, entity type, date range, uploaded by. Shows which record each document belongs to with a click-through link. Admin can find any document in the system from this one view.
STEP 4 — FILLABLE FORMS SYSTEM
Build these 11 forms as fully digital fillable forms. Every field on every government form must match the official form exactly. Every required field must be validated before submission — highlight missing fields in red, show a count of missing fields, block submission until complete. Every form must support digital signature capture (signature pad on mobile, draw with mouse on desktop).
W-4 (2024 version): All 5 steps including filing status checkboxes, multiple jobs worksheet, deductions worksheet, additional withholding. Employee fills all sections. Auto-calculates withholding amounts in Step 4.
I-9 (current version): Section 1 filled by employee (name, address, DOB, SSN, citizenship status, signature). Section 2 filled by HR (document verification — List A or Lists B and C, employer certification). Document upload fields for employee to attach photos of their ID documents (driver license, SSN card, birth certificate, or passport). HR must mark Section 2 complete separately after in-person verification.
Ohio IT-4: All fields matching the current Ohio Department of Taxation form. Employee fills and signs.
Direct Deposit Authorization: Bank name, routing number, account number, account type (checking/savings), authorization signature. Include a voided check upload option.
Employee Handbook Acknowledgment: Displays the current handbook version number and date. Employee confirms they have read it and signs. Auto-links to the current handbook document.
Emergency Contact Form: Up to 3 emergency contacts — name, relationship, phone (primary and secondary), address. Employee fills.
Background Check Authorization: Full legal authorization language compliant with the Fair Credit Reporting Act (FCRA). Employee name, SSN, DOB, address history for past 7 years, signature.
NDA (Non-Disclosure Agreement): Display the full NDA text. Employee prints name, signs, dates. Company representative signs separately.
Employment Application: Full application — personal info, work history (3 previous employers), education, references (3), skills, availability, how they heard about us, authorization to work statement, signature.
Workers' Comp First Report of Injury (Ohio BWC FROI): All fields matching the Ohio BWC First Report of Injury form. Section 1 filled by employee (injury description, body part, how it happened). Section 2 filled by supervisor/manager (witness info, medical treatment, return to work status). Routes to supervisor for Section 2 completion after employee submits Section 1.
OSHA Incident Report (OSHA 301): All fields matching OSHA Form 301. Employee fills their sections, supervisor completes employer sections. Links to the injured employee's record automatically.
STEP 5 — FORM WORKFLOW AND PDF GENERATION
Form assignment: HR or Admin can assign any form to an employee from the employee's profile — click Assign Form, select the form type, employee receives a notification in CompanyHQ and an email with a link to complete it. Forms appear in the employee's CompanyHQ dashboard under Action Required.
Form completion: Employee opens the form, fills all fields, signs digitally, submits. The form validates all required fields before allowing submission. If anything is missing it highlights the fields and shows exactly what needs to be filled.
PDF generation: On submission, generate a PDF that exactly replicates the form layout with all filled values and the digital signature embedded. Use a PDF generation library (pdfkit or puppeteer). Save the PDF to the documents system with home_entity_type = employee and home_entity_id = that employee's id. Category = form.
Onboarding checklist integration: Each form type maps to an onboarding checklist item. When a form PDF is successfully generated and saved, automatically mark the corresponding onboarding checklist item as complete. The checklist item cannot be manually checked off — it only completes when the real form is submitted.
HR notification: When any form is submitted, send an email notification to HR and Admin: employee name, form type, submission time, link to view the completed PDF in CompanyHQ.
STEP 6 — SHARE EXTERNALLY FEATURE
The Share Externally button appears on every document in the system. Admin role only. Clicking it opens a modal with: Expiration selector (24 hours, 7 days, 30 days, custom date), optional password field with strength indicator, optional note/message field (shown to recipient), Generate Link button. After generating: display the full URL in a copyable text box, Copy Link button that copies to clipboard, option to send directly via email by entering an address. The public share page at /share/:token requires no login. It checks token validity and expiry, prompts for password if set, displays the document with the admin's note, shows expiration date, has a Download button. Logs every access with timestamp and IP. In Admin Panel Documents section add a Shared Links manager: table of all active links with document name, created by, expiry, view count, last accessed, Revoke button. Expired and revoked tokens show a clean branded expiration page."
