# Chapin Landscapes — Daily Worksheet App: Complete Build Spec
**Prepared for the Replit developer.**
This document contains everything needed to build the Daily Worksheet feature: form structure, field definitions, validation rules, AI voice assistant behavior, and PDF export mapping. No other files are required to understand the spec — the PDF template (`Daily_Worksheet_Fillable.pdf`) is only needed at export time.

---

## 1. Overview & How This Form Works

The Daily Crew Worksheet is filled out once per day by the crew foreman, either by tapping through fields in the app or by talking to the AI voice assistant. Here is how the system should behave:

- **Validation lives in the app.** The PDF template has field flags set, but the real enforcement is in your UI. Block form submission if required fields are empty. Show inline errors next to each missing field.
- **The PDF is export-only.** When the user taps "Export" or "Print," the app fills the PDF template programmatically and either downloads it or opens a print dialog. The PDF is not the primary UI.
- **The AI assistant fills fields conversationally.** Users will say things like "the customer is Smith Residence" or "we arrived at 7:30." The AI maps those phrases to the correct field. Voice prompts for every field are listed in the schema below.
- **Required fields must be confirmed at the start of every session** before the user fills anything else. See the Session Start Flow in Section 5.

---

## 2. Required vs. Optional Fields

| Field | Required | Notes |
|---|---|---|
| Customer Name | **YES** | Block submission if empty. Highlight with error. |
| Date | **YES** | Block submission if empty. Default to today. |
| All other fields | No | Variable day to day. Never block submission for these. |

**Validation rule:** When the user taps Submit, check `customer_name` and `date`. If either is empty, show an inline error message directly under that field: *"This field is required before submitting."* Do not use a modal alert. Do not clear or reset other fields.

## 4. Computed Fields (Auto-Calculate — No User Input Required)

| Field | Computed From | Logic |
|---|---|---|
| `day_of_week` | `date` | `new Date(date).toLocaleDateString('en-US', {weekday: 'long'})` |
| `foreman_total_hours` | `foreman_arrival_time` + `foreman_departure_time` | Departure minus arrival in decimal hours. Example: 7:30 AM → 4:00 PM = 8.5 hours |
| `team_member_{n}_total_hours` | `team_member_{n}_arrival_time` + `team_member_{n}_departure_time` | Same logic as foreman |

Allow the user to manually override any computed field if needed.

## 5. AI Voice Assistant — Behavior & Session Flow

### Session Start Flow
Run this sequence at the beginning of every new worksheet session before letting the user fill other fields freely:

1. Auto-fill today's date. Confirm with user: *"I've set today's date to [date]. Does that look right?"*
2. Ask: *"Who is the customer for today's job?"* → maps to `customer_name`
3. Ask: *"What's the job site address?"* → maps to `address_line_1` (user can say "skip" to move on)
4. Ask: *"Who's on the crew today?"* → fill `foreman_name` and `team_member_{n}_name` in order
5. After that, let the user drive via voice or by tapping through fields freely.

### Ambiguous Voice Command Rules

| If the user says... | Do this |
|---|---|
| "set customer" or "the customer is" | → `customer_name` |
| "we arrived at [time]" with no name | Apply to foreman by default. Then ask: *"Just you, or the whole crew?"* |
| "add a task" or "we did" | Create the next available `work_item` row |
| A chemical name partially (e.g., "Dimension" or "Roundup") | Fuzzy-match to the closest pre-loaded chemical in the chemical log |
| "add to punch list" or "needs to be fixed" | Append to the next available `punch_item` |
| "we used the skid steer" or "skid steer was running" | Map to `equipment_log` → Skid Steer row |
| Time in any format (7:30 AM, 07:30, 7:30) | Normalize to HH:MM AM/PM before storing |

### Token Substitution
In the voice prompt arrays, `{n}` means the instance number (1, 2, 3…), `{name}` means the team member's actual name once it has been set, and `{chemical_name}` / `{equipment_name}` means the pre-loaded label for that row. Resolve these tokens at runtime so the AI can match "John arrived at 8" to Team Member 2's arrival time if John is Member 2.
