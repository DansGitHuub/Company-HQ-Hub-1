Please build a Reports module at /reports. This is the Data Center for the business — landscape company owners live in these numbers.

Page structure: A /reports page with sidebar tab navigation (left side tabs, similar to the Settings page layout) containing these 4 report sections:

---

1. Revenue Report
- Filters at top: Date Range (this month / last month / this quarter / last quarter / this year / custom), Division (All / Maintenance / Install / Snow / General), Salesperson (All + list of users)
- Summary cards row: Total Revenue, # of Jobs, Avg Job Value, % vs Prior Period
- Bar chart: Revenue by Month (12 months rolling) using Recharts
- Breakdown table: Division | Jobs | Revenue | % of Total

2. Job Costing Report
- Filters: Date Range, Division, Status (All / In Progress / Completed)
- Summary cards: Total Estimated, Total Actual Cost, Gross Profit, Gross Margin %
- Table: Job Name | Customer | Division | Est. Revenue | Est. Hours | Actual Hours | Variance | Status
- Pull from jobs table (use whatever columns exist, mock anything missing)

3. Invoice Aging Report
- As-of date picker (default today)
- Summary cards: Total Outstanding, Current (0-30 days), 31-60 Days, 61-90 Days, 90+ Days
- Table: Invoice # | Customer | Invoice Date | Due Date | Amount | Days Outstanding | Status
- Color-code rows: green=current, yellow=31-60, orange=61-90, red=90+
- Pull from invoices table

4. Crew Hours Report
- Filters: Date Range, Employee (All + list), Division (All + list)
- Summary cards: Total Hours, Regular Hours, Overtime Hours (>40/week), Avg Hours/Employee
- Bar chart: Hours by Week (last 12 weeks) using Recharts
- Table: Employee Name | Role | Regular Hrs | OT Hrs | Total Hrs | Jobs Worked

Tech notes:
- Use Recharts for charts (already installed)
- Create API endpoints: GET /api/reports/revenue, /api/reports/job-costing, /api/reports/invoice-aging, /api/reports/crew-hours
- Each endpoint aggregates real data using pool.query(). If a column doesn't exist yet, return mock placeholder data so the UI still renders beautifully
- Use shadcn/ui Card and Table components matching the rest of the app's style
- Admin/Manager roles see all data; regular users see only their own
- Add route in App.tsx: <Route path="/reports" component={ReportsPage} />
- The /reports sidebar link should now route to this new page
