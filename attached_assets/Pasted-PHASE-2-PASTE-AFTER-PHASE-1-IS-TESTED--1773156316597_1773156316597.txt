TOOL LIBRARY — ADD ALL OF THESE
Add these tools to the /api/assistant/chat endpoint. Each tool definition needs: name, description (used by GPT to know when to call it), parameters schema (JSON Schema format). Each tool must validate user permissions before executing. All tools call existing API routes — never bypass the existing route handlers.
navigateTo(module: string, subpage?: string) — already exists from Phase 1.
openRecord(type: 'equipment'|'task'|'employee'|'job', id: string) — navigate to a specific record's detail page.
searchGlobal(query: string) — full text search across tasks, equipment, employees, SOPs, messages. Returns top 5 results from each category.
searchEquipment(filters: { status?, category?, assignedTo?, maintenanceDue? }) — returns matching equipment records with their current priority status.
searchTasks(filters: { assignedTo?, priority?, status?, dueToday? }) — returns matching task records.
searchEmployees(query: string) — search employees by name, role, or department.
searchSOPs(query: string) — full text search across SOP library.
getDailyBriefing() — returns a structured summary: overdue tasks (count + titles), urgent equipment (count + names), unread messages count, tasks due today count. GPT formats this into a natural language morning summary.
createTask(title: string, assignedToUserId: number, priority: 'p1'|'p2'|'p3'|'p4', dueDate?: string, description?: string) — creates a task. REQUIRES ActionPreviewCard confirmation before executing.
updateTaskStatus(taskId: string, status: string, note?: string) — updates a task status. Confirm before executing for complete/cancelled.
createEquipment(data: { assetName, category, make, model, year?, serialNumber?, assignedTo?, currentHours? }) — creates a new equipment asset. REQUIRES ActionPreviewCard confirmation.
logEquipmentService(assetId: string, serviceData: { taskName, serviceDate, hoursAtService, performedBy, notes?, cost? }) — logs a completed maintenance task. REQUIRES confirmation.
updateEquipmentHours(assetId: string, currentHours: number) — updates current hours on an asset. Confirm before executing.
lookupVIN(vin: string) — calls NHTSA API, returns decoded vehicle info.
submitRepairRequest(assetId: string, description: string, severity: 'minor'|'moderate'|'major', isUsable: string) — submits repair request. REQUIRES confirmation.
CONFIRMATION FLOW
For any tool marked REQUIRES confirmation: before calling the tool, GPT must return a structured response with type: 'confirmation_required' and a summary of what will happen. The frontend displays this as an ActionPreviewCard with Confirm and Cancel buttons. Only when the user taps Confirm does the frontend send a follow-up message { confirmationGranted: true, pendingTool: {...} } and the backend executes the tool. If the user taps Cancel, send { confirmationGranted: false } and the assistant acknowledges the cancellation. This entire flow must be reliable — never execute a write operation without receiving explicit confirmation.
PERMISSION ENFORCEMENT
The Tool Executor validates permissions server-side before executing any tool, regardless of what the AI requested. Use the same permission rules as the rest of the app: role-based access, record-level access (crew lead can only update equipment assigned to their crew), etc. If permission fails, return a structured error — the AI will explain it to the user in plain English. Never expose raw permission errors or database errors to the user.
