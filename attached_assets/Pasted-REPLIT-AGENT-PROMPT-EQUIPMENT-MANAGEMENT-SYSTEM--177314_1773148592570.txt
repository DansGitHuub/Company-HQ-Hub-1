REPLIT AGENT PROMPT — EQUIPMENT MANAGEMENT SYSTEM
──────────────────────────────────────────────────────────
Rebuild the Equipment module in CompanyHQ into a complete automated fleet management system. Read everything before writing code.
OVERVIEW
The Equipment module tracks every company asset (mowers, tractors, trucks, trailers, handheld tools), automates maintenance scheduling based on engine hours AND calendar time, sends priority-based alerts, logs all service and repairs, stores all documents and receipts, and provides a fleet health dashboard. The core innovation is the OEM maintenance library — pre-loaded manufacturer schedules that auto-assign to equipment when it's added.
SECTION 1: DATABASE SCHEMA
Create these tables using Drizzle ORM:
equipment_assets — id, asset_name, nickname, asset_id (auto EQ-XXXX), category, make, model, year, serial_number, purchase_date, purchase_price, purchased_from, condition_at_purchase, status (active/in_repair/stored/retired/sold), assigned_to_user_id, primary_location, tracking_type (hours/miles/season), current_hours, hours_at_purchase, last_hours_update, license_plate, registration_expiry, insurance_expiry, warranty_expiry, primary_photo_url, created_at, updated_at
oem_maintenance_templates — id, brand, category, task_name, task_description, hours_interval (nullable), calendar_interval_days (nullable), priority_level, notes — this is the library of pre-loaded manufacturer schedules
equipment_maintenance_schedules — id, asset_id, template_id (nullable), task_name, task_description, hours_interval, calendar_interval_days, last_service_hours, last_service_date, next_due_hours, next_due_date, priority (p1/p2/p3/p4), is_overridden, override_notes, is_active
service_log — id, asset_id, log_type (scheduled/repair/inspection/hours_update), task_name, service_date, hours_at_service, performed_by_user_id, service_location, parts_used (JSON array: part_number, description, qty, unit_cost), labor_cost, total_cost, receipt_url, notes, created_by, created_at
repair_requests — id, asset_id, reported_by_user_id, report_date, problem_description, severity (minor/moderate/major), is_usable (yes/restricted/no), photos (JSON array of URLs), status (open/in_review/in_repair/resolved), assigned_to_user_id, shop_name, drop_off_date, expected_return, resolution_description, resolution_date, total_repair_cost, receipt_url
equipment_documents — id, asset_id, folder (manuals/purchase/warranty/registration/insurance/receipts/photos/other), file_name, file_url, file_type, uploaded_by, uploaded_at, notes
SECTION 2: OEM MAINTENANCE LIBRARY
Seed the oem_maintenance_templates table with pre-loaded schedules for these brands: Exmark (commercial mowers), Kubota (compact tractors), Stihl (handheld power tools), Ford/GM/Ram (trucks — use generic manufacturer-recommended intervals), and generic trailer maintenance. Include all service tasks with hours_interval and calendar_interval_days values. When a user adds an asset and selects make + category, automatically query the template library and create equipment_maintenance_schedules records for that asset from the matching templates. Allow per-asset overrides without changing the library.
SECTION 3: PRIORITY CALCULATION ENGINE
Build a server-side function that recalculates priority for every maintenance schedule item whenever: (a) hours are updated for an asset, (b) a service is logged completing a task, (c) a daily cron job runs at 6am. Priority logic: P4 = not approaching. P3 = within 10% of hours interval remaining OR within 14 calendar days. P2 = 0 hours remaining OR within 7 calendar days. P1 = negative hours (overdue) OR past due date. Store calculated priority on the equipment_maintenance_schedules record. Also calculate next_due_hours = last_service_hours + hours_interval, and next_due_date = last_service_date + calendar_interval_days.
SECTION 4: NOTIFICATION SYSTEM
Build these automated notifications using the existing notification/email infrastructure: (1) Daily digest at 7am to managers: all P1 and P2 items across all equipment, grouped by asset. (2) Immediate push + email to manager AND asset's assigned_to user when any item becomes P1. (3) Weekly Monday summary to manager: all P3 items. (4) Registration/insurance expiry: email at 60 days, 30 days, 7 days before. (5) When a service is logged as complete: in-app notification to manager confirming completion. (6) Repair request submitted: immediate notification to manager. (7) If repair request is not reviewed within 24 hours: escalation email to manager.
SECTION 5: EQUIPMENT DASHBOARD
Build a dashboard landing page for the Equipment module with: (1) Fleet health summary tiles: total active assets, P1 count (red), P2 count (orange), P3 count (yellow), in-repair count, compliance alerts count. Each tile is clickable and filters the list below. (2) Equipment list with columns: asset ID, name, category, assigned to, status, health indicator dot, next service due (task + days/hours), last hours update. Sortable columns. (3) Quick action buttons per row: Update Hours, Log Service, Report Repair. (4) Maintenance Queue tab: all pending tasks across all equipment sorted P1 → P2 → P3, with Mark Complete button on each.
SECTION 6: ASSET PROFILE PAGE
Each asset has a full profile page with tabs: (1) Overview — all identity fields, photo, current status, registration/insurance/warranty expiry dates with color-coded indicators. Edit button for managers. (2) Maintenance Schedule — list of all maintenance tasks for this asset, each showing task name, interval, last done (date + hours), next due (date + hours), priority badge, Mark Complete button. (3) Service History — reverse chronological log of all service entries and repairs. Each entry expandable to show full details + receipt thumbnail. (4) Repair Requests — all open and resolved repair requests for this asset. Submit New Repair button. (5) Documents — folder-organized document storage with upload and download. (6) Hours Log — history of all hours/mileage updates with timestamp and entered by.
SECTION 7: SERVICE & REPAIR LOGGING FORMS
Service log form: date, current hours/miles, performed by (dropdown of staff + 'Outside Shop'), service location dropdown, parts used (repeating row: part number, description, qty, unit cost — auto-total), labor cost, total cost (auto-calculated), receipt upload (camera or file), notes. On submit: update last_service_hours and last_service_date on the schedule item, recalculate priority, create service_log record, attach receipt to equipment_documents. Repair request form: problem description, severity dropdown, is-usable dropdown, photo upload (multiple), submit. Manager review form: assign to user OR assign to shop (with shop name, drop-off date, return date). Resolution form: what was found, what was done, parts/cost, receipt upload, return to service date.
SECTION 8: ADD NEW EQUIPMENT FLOW
Multi-step form to add a new asset: Step 1 — category, make, model, year, serial number. Step 2 — auto-query OEM library for matching maintenance templates. Show the user: 'We found [N] maintenance tasks for this equipment. These will be added to its schedule automatically.' Allow them to review and deselect any. If no template exists, prompt: 'No template found. You can add a custom schedule after saving, or we can help you create one.' Step 3 — remaining fields: purchase info, registration, insurance, assigned to, location. Step 4 — upload documents (manual, purchase receipt, etc.) — all optional, can be done later. Step 5 — set current hours and hours at purchase (for used equipment). On save: create asset record + all maintenance schedule records from templates.
SECTION 9: CALENDAR INTEGRATION
Add equipment maintenance events to the CompanyHQ calendar module. Each calendar-based maintenance task generates a calendar event on its next_due_date. P1 events show in red, P2 in orange, P3 in yellow. Add filters to the calendar: Equipment Only view, filter by category, filter by priority. Events are clickable and open the maintenance task. When a service is marked complete, remove the current event and generate the next one at the new next_due_date.
SECTION 10: VIN DECODE FOR VEHICLES & TRAILERS
Add VIN decoding to the Add New Equipment flow for any asset with category = Truck, Vehicle, or Trailer. When the user enters a VIN number, call the free NHTSA VIN decode API at https://vpic.nhtsa.dot.gov/api/vehicles/decodevin/[VIN]?format=json (no API key required). Auto-populate these fields from the response: Make, Model, Year, Body Class, Engine Cylinders, Engine Displacement, Fuel Type, Transmission Type. Display the decoded results to the user and let them confirm or edit before saving — never auto-save without confirmation. After confirmation, use the decoded Make + Category to query the oem_maintenance_templates table and auto-assign the matching maintenance schedule, exactly as the standard add-equipment flow does. Store the raw VIN string on the asset record. If the NHTSA API returns no results, returns an error, or times out, fail gracefully with a friendly message ('VIN lookup unavailable — please fill in the details manually') and let the user complete the form by hand. Do not block the form submission on a failed VIN lookup.
SECTION 11: ROLES & PERMISSIONS
Admin and Manager: full access. Crew Lead: can view all equipment, log service and update hours on assigned equipment, submit repair requests. Cannot add assets or modify maintenance schedules. Crew Member: can view assigned equipment basic info, update hours, submit repair requests only. HR/Sales/New Hire: no access to this module.
SECTION 12: BUILD ORDER
1. Database schema and migrations. 2. OEM maintenance library seed data. 3. Add equipment flow with auto-template assignment and VIN decode. 4. Priority calculation engine + cron job. 5. Equipment dashboard and asset list. 6. Asset profile page with all tabs. 7. Service log and repair request forms. 8. Document upload and storage. 9. Notification triggers. 10. Calendar integration. 11. Role-based access control. Do not use mock data — all data must persist to the real database.
